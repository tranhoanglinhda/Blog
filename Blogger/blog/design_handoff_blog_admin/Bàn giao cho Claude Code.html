01 · Tổng quan

Sản phẩm là gì

Một blog cá nhân theo phong cách tản văn/nhật ký (lấy cảm hứng từ blog gốc của bạn) gồm 2 phần: trang công khai cho người đọc, và trang quản trị có mật khẩu để tác giả viết, sửa, đăng bài. Toàn bộ giao diện đã được thiết kế trong file prototype — Claude Code chỉ cần dựng lại bằng code thật và gắn cơ sở dữ liệu.

🌿 Trang công khai

  • Trang chủ: bài nổi bật + danh sách + sidebar
  • Trang chi tiết bài viết (typography editorial)
  • Trang theo chủ đề / tag
  • Trang giới thiệu tác giả (About)
  • Tìm kiếm tức thời (overlay)
  • Bình luận của khách (không cần đăng nhập)
  • Sidebar: tiểu sử, bài đọc nhiều, chủ đề, lưu trữ

⚙️ Trang quản trị

  • Đăng nhập (email + mật khẩu)
  • Tổng quan: thống kê + bài gần đây
  • Quản lý bài: lọc theo trạng thái, sửa, xóa
  • Trình soạn thảo rich-text + thanh công cụ
  • Ảnh bìa, slug, mô tả ngắn, chủ đề
  • Lưu nháp / Xuất bản · Cài đặt blog

03 · Mô hình dữ liệu

Data model

Post {
  id            string (uuid)
  slug          string  unique        // "tuoi-ham-hai"
  title         string
  excerpt       string                // mô tả ngắn, hiện ở feed
  body          richtext (HTML/JSON)  // nội dung bài
  coverImageUrl string?               // ảnh bìa đã upload
  coverHue      int                   // tông màu placeholder (prototype)
  status        enum(draft|published)
  tags          Tag[]                 // quan hệ nhiều-nhiều
  readMins      int                   // tự tính từ độ dài
  views         int     default 0
  featured      bool    default false
  authorId      string  -> User
  publishedAt   datetime?
  createdAt     datetime
  updatedAt     datetime
}

Tag  { id, slug unique, name }            // "gia-dinh" -> "Gia đình"

Comment {                                  // bình luận của khách
  id        string
  postId    string  -> Post
  parentId  string?  -> Comment                  // null = gốc; có giá trị = trả lời (lồng 1 cấp)
  name      string                              // tên khách (bắt buộc)
  email     string?                             // tùy chọn, không hiển thị
  text      string
  likes     int     default 0                   // lượt tim
  status    enum(visible|pending|spam)          // để kiểm duyệt
  createdAt datetime
}
// Lượt tim của khách ẩn danh: nên có bảng CommentLike(commentId, visitorId)
// để chống tim trùng; prototype lưu cờ liked phía client.

User { id, email unique, passwordHash, name, bio, avatarUrl }

SiteSettings { title, tagline, authorName, authorBio, contactEmail }
💡 Trong prototype, coverHue sinh ảnh bìa gradient giả lập. Khi dựng thật, thay bằng upload ảnh thực (coverImageUrl) và giữ coverHue làm màu nền dự phòng khi chưa có ảnh.
04 · Tính năng admin chi tiết

Yêu cầu chức năng

Trình soạn thảo

Rich-text WYSIWYG: đậm, nghiêng, tiêu đề (H2), trích dẫn, danh sách, liên kết, chèn ảnh. Tiêu đề là ô riêng. Gợi ý dùng Tiptap hoặc Lexical thay cho contentEditable thô của prototype.

Nháp & Xuất bản

Mỗi bài có trạng thái draft/published. Nút Lưu nháp và Xuất bản. Nên có tự lưu (autosave) và xác nhận trước khi rời trang.

Upload ảnh bìa

Tải ảnh lên kho (S3 / Cloudinary / UploadThing). Hỗ trợ chú thích ảnh. Sinh thumbnail cho feed.

Xác thực

Đăng nhập 1 tài khoản tác giả là đủ (single-author). Bảo vệ mọi route /admin/*. Gợi ý: Auth.js (NextAuth) hoặc Lucia.

Bình luận của khách

Khách bình luận cuối mỗi bài (tên + nội dung, email tùy chọn), không cần đăng nhập. Có trả lời lồng nhau (1 cấp) và lượt tim cho cả bình luận lẫn reply. Avatar sinh từ chữ cái đầu. Kiểm duyệt trong admin: tab Chờ duyệt / Hiển thị / Spam, thao tác Duyệt · Ẩn · Spam · Xóa, kèm chống spam (honeypot/rate-limit).

05 · Hệ thống thiết kế

Design tokens

Màu (mặc định — “clay”)

--accent #BC5B3A
--accent-ink #9C462A
--accent-soft #F0DDD2
--paper #FBF7F1
--ink #26211B
--hairline #E7DDCE

Bản prototype có thêm 4 bảng màu thay thế (sage, indigo, amber, plum) chỉnh được qua panel Tweaks — dùng làm gợi ý chủ đề có thể chọn.

Kiểu chữ

Noto Serif (serif)Tiêu đề & thân bài. Văn chương, hỗ trợ tiếng Việt đầy đủ (precomposed). Lora / Spectral là lựa chọn thay thế.
Be Vietnam Pro (sans)Giao diện, meta, nút, admin. Thiết kế riêng cho tiếng Việt.

Quy ước khác

Bo góc thẻ: 12px Bo góc nút: 9px Đo dòng bài viết: ~680px Khung trang: 1180px Bóng nhẹ, hairline ấm Không emoji trong UI thật
Tất cả token nằm trong blog/styles.css dưới dạng biến CSS --* — copy trực tiếp sang dự án thật.
06 · Gợi ý kỹ thuật & file

Bàn giao

Stack đề xuất

  • Next.js (App Router) + TypeScript
  • Postgres + Prisma (hoặc Supabase)
  • Auth.js cho đăng nhập tác giả
  • Tiptap/Lexical cho rich-text
  • Cloudinary/UploadThing cho ảnh
  • Tailwind (map theo token ở trên)

File trong prototype

  • Sổ Tay Của Mây.html — bản chạy được
  • styles.css — design tokens
  • public.jsx — UI trang công khai
  • admin.jsx — UI trang quản trị
  • data.js — dữ liệu mẫu (cấu trúc Post)
  • helpers.jsx — icon, ảnh bìa, format
